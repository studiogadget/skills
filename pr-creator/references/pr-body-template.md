# PR本文テンプレート

PR本文を生成する際、このテンプレートの構造に従うこと。

---

## 冒頭: AIレビュールールセクション（必須・変更禁止）

以下のブロックをPR本文の**先頭にそのまま**含める。一切省略・変更しないこと。

```markdown
<!-- I want to review in Japanese. -->
<!-- Review rules for AI -->

<details>
<summary>Review rules for AI</summary>

### お願い

- Pull Request Overview、Reviewed Changes、コメントはすべて必ず日本語で書いてください

### レビュー対象と範囲

- 次の観点でコード品質の改善提案を行ってください
  - 設定ファイルの適切な利用（パラメータのハードコーディングなく、設定ファイルから読み込んだ値を使用しているか。呼び出し元で設定値が渡されている場合も含む）
  - ロジックの正確性
  - セキュリティー上の問題
  - パフォーマンスの最適化
  - データ競合の可能性
  - 一貫性の維持
  - クラス名、メソッド名、フィールド名に表記ゆれが無いこと
  - エラーハンドリング
  - 保守性の向上
  - モジュール性の確保
  - コードの複雑性軽減
  - 最適化の可能性
  - ベストプラクティス(DRY, SOLID, KISS)の適用
- 重大な問題(例:テスト失敗、脆弱性、O(n²) 以上のボトルネック)に絞ってレビューしてください
- 次についてはコメントしないでください
  - `db/**`, `**/*.lock`
  - 自動生成されたコード(l10n, freezedなど)
  - コードレビュー結果ファイル

</details>
<!-- Review rules for AI -->
<!-- I want to review in Japanese. -->
```

## 本文構成

AIレビュールールセクションの後に、以下の構成で本文を記載する。

### 概要

```markdown
## 概要

### WHAT（何を変更したか）
- <主要変更点を箇条書き>

### WHY（なぜこの変更が必要か）
- <背景・課題>

### IMPACT（どのような影響があるか）
- <影響範囲の要約>
```

### 変更内容

```markdown
## 変更内容

- <主要変更点を箇条書き>

### 変更の種類
- [ ] 機能追加
- [ ] バグ修正
- [ ] リファクタリング
- [ ] ドキュメント
- [ ] テスト
- [ ] その他
```

### テスト

```markdown
## テスト

<!-- いずれか1つを選択して記載 -->

<!-- テスト実装済みの場合 -->
- <テスト内容を記載>

<!-- テスト不要の場合（ドキュメント更新など） -->
- テスト不要（理由: <理由>）

<!-- テスト未実装の場合 -->
- [ ] <実装すべきテスト1>
- [ ] <実装すべきテスト2>
```

### 影響範囲

```markdown
## 影響範囲

- 影響を受けるモジュール/機能: <記載>
- 後方互換性: あり / なし（なしの場合は詳細を記載）
```

### 確認項目

```markdown
## 確認項目

- [ ] コードの品質チェック完了
- [ ] テスト追加/修正済み
- [ ] 破壊的変更なし or BREAKING明記
```
