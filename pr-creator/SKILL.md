# PR Creator

Git運用規則に準拠したドラフトPRを作成するスキル。

## 厳守事項（Hard Rules）

**作業開始前に以下4点を必ず復唱すること：**

1. **テンプレート保持**: PR本文冒頭のAIレビュールールセクションを完全にそのまま含める（省略・変更禁止）
2. **PR原則**: 1PR = 1目的、差分400行以内（テスト除く）
3. **ドラフト形式**: 必ず `--draft` オプション付きで作成
4. **本文構成**: 概要（WHAT/WHY/IMPACT）→ 変更内容 → テスト → 影響範囲 → 確認項目の順

**Git操作許可範囲**: `gh pr create --draft` のみ実行可（コミット・プッシュ禁止）

## 入力

| パラメーター | 既定値 | 説明 |
|---|---|---|
| base | `main` | ベースブランチ |
| compare | 現在のブランチ | 比較ブランチ |
| issue | なし | 関連Issue番号 |

## 手順

### Step 1: 差分分析

1. `git diff <base>...<compare> --stat` で変更ファイルと行数を確認
2. 差分が400行超過（テスト除く）の場合 → 警告を表示（PR作成は続行可能）
3. `git log <base>..HEAD --oneline` でコミット履歴を確認し、WHAT/WHYを抽出

### Step 2: PR本文生成

[PR本文テンプレート](./references/pr-body-template.md) を読み込み、以下のルールで本文を生成する。

#### タイトル

- 日本語簡潔、72文字以内
- 「ドラフト」「WIP」等のプレフィックスは付けない
- コミットメッセージと変更内容から自動生成

#### 本文

1. **最優先**: [テンプレート](./references/pr-body-template.md)冒頭のAIレビュールールセクションをそのまま含める
2. 概要（WHAT / WHY / IMPACT）を記載
3. 変更内容を箇条書きで記載、変更種類にチェック
4. テスト内容を記載（未実装ならチェックボックス付きTODO）
5. 影響範囲を記載（後方互換性の有無含む）
6. 確認項目のチェックボックスはすべて未チェック状態で残す
7. 関連Issueがあれば `Closes #<issue>` を追加

### Step 3: PR作成

確認なしで即座に実行する（ユーザーへの提示・確認は不要）。

```bash
gh pr create --draft --base <base> --head <compare> --title "<title>" --body "<body>"
```

### 完了時の出力

```
**PR作成結果**:
- PR番号: #<PR番号>
- PR URL: <URL>
- タイトル: <タイトル>
- 実行コマンド: [再現用コマンド]
```

## エラーハンドリング

| 状況 | 対応 |
|---|---|
| 差分400行超過 | 警告表示、分割提案（PR作成は続行可） |
| ブランチ名がGit規則に非準拠 | 警告表示（PR作成は続行可） |
| 関連Issueが見つからない | 警告表示（PR作成は続行可） |
