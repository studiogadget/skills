# スライド作成スキル（pptx-slides-creator）

ユーザーが提供した資料、または指示に従って調査した結果から、トップティア戦略コンサルティングファーム（McKinsey/BCG）品質の情報密度の高い日本語プレゼンテーションを作成する。テックミニマリストかつ情報リッチなビジュアルスタイルで、高級エディトリアルデザインの美学を融合させる。

## トリガー条件

- ユーザーが生データ、メモ、リサーチ、ドキュメントを提供し、プロフェッショナルなスライドを求めた場合
- ユーザーがトピックを指定し、調査結果をスライドにまとめるよう依頼した場合
- 「McKinseyスタイル」「BCGスタイル」「コンサルスライド」「戦略プレゼン」などの要求
- 経営ブリーフィング、取締役会資料、投資家向けプレゼンテーション
- チャートやフレームワーク、テーブルを含む高密度の分析スライドの要求

## 前提条件

作業開始前に、必ず以下のリファレンスを順番に読み込むこと：

1. **デザインシステム**: `pptx-slides-creator/references/design-system.md` — カラー、フォント、レイアウト定数、全コンポーネントのコード
2. **スライドパターン集**: `pptx-slides-creator/references/slide-patterns.md` — スライド構成パターンとデータに応じたレイアウト選択基準
3. **テンプレート**: `pptx-slides-creator/scripts/template_deck.js` — 新規デッキ作成時のスターターコード

## ワークフロー（5ステップ）

### Step 1: 素材の取得と分析

**資料が提供された場合:**
アップロードされたファイルまたはテキストから以下を抽出する：
- **キーメッセージ**: 主要な2〜3個の結論・洞察
- **データポイント**: 数値、パーセンテージ、トレンド、比較データ
- **フレームワーク候補**: SWOT、2x2マトリクス、プロセスフロー、ファネル、タイムライン
- **ナラティブアーク**: 全体の物語構造（問題→分析→示唆）

**トピック指定で調査が必要な場合:**
1. Web検索で最新の公開情報・データを収集する
2. 信頼性の高いソース（公式発表、調査レポート、業界データ）を優先する
3. 収集した情報を上記の4観点で整理する
4. データソースを記録する（スライドの出所引用に使用）

### Step 2: デッキ構成の計画

コンサルティングデッキは厳密なナラティブロジックに従う。`references/slide-patterns.md` を参照し、データ特性に合ったレイアウトを選択する。

**標準構成（6〜8枚）:**
1. **タイトルスライド** — テーマ、サブタイトル、日付。ダークネイビー背景。
2. **エグゼクティブサマリー** — 「So What（要は何か）」を先に提示。KPIコールアウト＋重要発見事項。
3. **状況 / コンテキスト** — 背景データ、市場概況、トレンドチャート。
4. **分析スライド**（2〜4枚）— 核となる分析。以下を組み合わせる：
   - データビジュアライゼーション（チャート＋注釈）
   - フレームワーク（2x2マトリクス、比較テーブル、プロセスフロー）
   - 深堀り（マルチカラム＋補足データ）
5. **示唆 / 提言** — 構造化されたテイクアウェイ、優先度付きアクション。
6. **付録**（任意）— 補足データテーブル、方法論ノート。

**必須ルール: アクションタイトル**

各スライドのタイトルは、トピックラベルではなく**洞察を述べる文**にする：
- ✅ 「デジタル売上は前年比34%増、エンタープライズ契約が主因」
- ❌ 「売上概要」

### Step 3: デザインシステムに従ったスライド構築

`references/design-system.md` のデザインシステムに**厳密に**従う。以下は要約：

**カラー:** 白背景(`FFFFFF`)、ディープネイビー見出し(`0B1F3F`)、ロイヤルブルーチャート(`1B3A6B`)、ブルー→グレーのグラデーション、アクセントレッド(`C8102E`)は控えめに

**タイポグラフィ:** 見出しは游明朝（セリフ体でプレミアム感）、本文は游ゴシック（サンセリフ体で可読性）、数値はCalibri

**レイアウト:** 全辺0.5インチマージン、マルチカラム推奨、タイトル下にネイビー0.5ptライン、左下にソース引用、右下にページ番号

### Step 4: チャート・テーブル・フレームワークの実装

`references/design-system.md` のコンポーネントコードを使用する。主要パターン：

| データ特性 | 推奨チャート | 備考 |
|-----------|------------|------|
| カテゴリ比較 | 棒グラフ（カラム） | ロイヤルブルー単色 or ブルー系グラデーション |
| 構成比・内訳 | 積み上げ棒グラフ / ドーナツ | セグメント5つまで |
| 時系列推移 | 折れ線グラフ | 2ptライン、マーカー付き |
| 増減要因分析 | ウォーターフォール | 増加=ブルー、減少=レッド |
| 定量比較 | データテーブル | ヘアラインボーダー、交互行背景 |
| 戦略的位置付け | 2x2マトリクス | シェイプで構築、画像不可 |
| プロセス・手順 | プロセスフロー | 矢印付きステップ |
| 段階的絞り込み | ファネル | 台形シェイプの積み上げ |
| 項目対比 | Before/After比較 | 2カラムレイアウト |
| 時系列イベント | タイムライン | ベースライン＋マイルストーン |

**データが明示的に提供されていない場合:** 資料からトレンドが暗示される場合は代表的なビジュアライゼーションを作成し、「図示のため仮定値を使用」と注記する。

### Step 5: QA・検証

1. スクリプトを実行してPPTXを生成する
2. 全スライドを検査し、以下を確認する：
   - [ ] テキストのオーバーフロー・重なりがないか
   - [ ] チャートの可読性（ラベル、凡例、値）
   - [ ] マージンとアライメントの一貫性
   - [ ] カラーパレットの一貫性（定義外の色を使っていないか）
   - [ ] 全コンテンツスライドにアクションタイトルがあるか
   - [ ] 全データスライドにソース引用があるか
   - [ ] 日本語テキストの切れ・文字化けがないか
3. 問題があれば修正して再生成する

## 日本語特有の注意点

- **フォント**: 見出し=游明朝（フォールバック: MS PMincho → Georgia）、本文=游ゴシック（フォールバック: Meiryo UI → Calibri）
- **文字サイズ**: 日本語は英語より1-2pt大きく設定して可読性を確保する
- **テキストボックス幅**: 日本語はゆとりのある幅設定が必須（英語比で1.2〜1.5倍）
- **句読点**: 全角「。」「、」を使用する
- **敬語**: ビジネス文書として「です・ます」調を基本とする
- **数字表記**: 漢数字と算用数字を適切に組み合わせる（例：1.2兆円、340億円）

## デザイン・アンチパターン（避けるべきこと）

- トピックラベルのみの汎用タイトル（アクションタイトルでないもの）
- ビジュアル要素のない箇条書きだらけのスライド
- 3つ以上のフォントファミリーの混在（游明朝 / 游ゴシック / Calibri の3種まで）
- ソース引用のないデータチャート
- ホワイトスペースのない詰め込みレイアウト
- 定義済みパレット外の明るい・彩度の高い色の使用
- 3D効果、グラデーション背景、ドロップシャドウなどの装飾
- 情報を伝えない装飾要素
- ビジュアル要素（チャート、テーブル、フレームワーク、コールアウト）が2つ未満のスライド
- 英語と日本語が混在する際のフォントサイズの不統一
