# Agent Skills

LLMエージェントで再利用できる「スキル（手順・プロンプト・ツール連携・検証観点）」を、**実運用で使える形**にパッケージ化して蓄積するためのリポジトリです。
個別プロジェクトの“その場しのぎ”を減らし、**同じ品質で何度でも使える**形に整備します。

> 目的：スキルを「コピペ可能なメモ」ではなく、**仕様・入力/出力・制約・テスト/評価**まで揃った“部品”として管理する。

---

## 収録するもの / 収録しないもの

### ✅ 収録するもの
- スキル定義（目的、前提、入出力、失敗パターン、ガードレール）
- 実行手順（プロンプト、ツール手順、チェックリスト）
- 最小のサンプル（例：入出力例、想定ケース）
- 評価（成功条件、簡易テスト、リグレッション観点）

### ❌ 収録しないもの（原則）
- 特定企業・特定案件の機密情報
- 個人情報、APIキー、トークン
- “一度しか使わない”プロジェクト固有の泥臭い手順（必要なら `projects/` 側へ）

---

## リポジトリ構成（使用時）

スキルを使用する際は `.github/skills/<skill_name>/` に配置して下さい。GitHub Copilot は、このディレクトリ配下をスキルのリポジトリとして認識し、必要に応じて使用します。

```
.
├─ .github/
│  └─ skills/
│     ├─ <skill_name>/
│     │  ├─ SKILL.md           # スキルの詳細（YAMLフロントマター + Markdown本文）
│     │  ├─ README.md          # （一部スキルに存在）
│     │  ├─ LICENSE.txt        # （一部スキルに存在）
│     │  ├─ NOTICE.md          # （一部スキルに存在）
│     │  ├─ assets/            # （一部スキルに存在）
│     │  ├─ references/        # （一部スキルに存在）
│     │  └─ scripts/           # （一部スキルに存在）
└─ LICENSE
```

---

## スキルの最低仕様（Quality Bar）

各 `.github/skills/<skill_name>/` は、最低でも以下を満たすことを推奨します。

- **YAMLフロントマター**：name, description必須
- **概要**：目的が1文で言える
- **作業手順**：明確なステップ（AIエージェント向けに簡潔に）
- **例（Examples）**：最低1ケース（入力例と期待出力）
- **同梱リソース**：必要に応じてreferences/, scripts/ など
- **チェックリスト**：評価観点（作成後の確認項目）

---

## スキル仕様フォーマット（例）

`.github/skills/<skill_name>/SKILL.md` の推奨テンプレ：

````markdown
---
name: <skill_name>
description: |-
  スキルの説明（トリガー条件、目的、流れの概要）
  例: Playwrightを使ったWebスクレイピングを実装する際に使用。
  「Webサイトからデータを抽出したい」といった依頼で起動。
  URL分析→データ抽出→保存処理の流れで進める。
---

# <Skill Name>

## 概要

スキルの目的と特徴を1-2文で説明。

## 作業手順

次の手順で進める（AIエージェント向けに簡潔に）:

1. ステップ1
2. ステップ2
3. ステップ3

## 例

**入力例**: 具体的な依頼例

**期待する出力**:
```text
// コード例
```

## 同梱リソース

以下のディレクトリに実用的なリソースを配置しています。

### references/
- 説明

### scripts/
- 説明

## 参照の使い方

- 使い方のヒント

## チェックリスト

作成後に以下を確認:

- [ ] description がトリガー条件を明確に示している
- [ ] SKILL.md が 200〜300行以内
- [ ] 手順に曖昧な表現がない
- [ ] コード例が実際に動く
````

---

## 使い方（最短）

1. 使用するスキルを `.github/skills/` 配下にディレクトリごと配置する（GitHub Copilotで使用する場合）
2. AIエージェント（例: GitHub Copilot）が自動的にスキルを認識し、必要に応じて使用する
3. 意図的に特定のスキルを使用したい場合は、SKILL.md の description に含まれるトリガー文字列をプロンプトに含めてAIエージェントに指示する

## スキル一覧

- **skill-creator**：GitHub Copilot用のスキル（Agent Skills）を新規作成・改修するガイドラインとテンプレート
- **skill-creator-from-logs**：VS Code / GitHub Copilot Chat の「Chat: Export Chat...」で出力した JSON を解析し、繰り返しワークフロー（WHAT/HOW/FLOW）を発見して Agent Skills を自動生成
- **playwright-scraper**：Playwrightを使ったWebスクレイピングの実装スキル
- **python-readable-code**：Pythonコードのリーダブルコード原則に基づくレビュー・改善スキル（AI駆動開発時代の「理解速度最大化」を重視）
- **storage-state**：Playwrightの storage_state.json を管理するスキル（生成スクリプト + 古い場合の警告ロジック）
- **pptx-slides-creator**：日本語のパワーポイントスライド（.pptx）を作成するスキル。McKinsey/BCGスタイルのコンサルティング品質で、データビジュアライゼーション、フレームワーク、テーブル、マルチカラムレイアウトを含む洗練されたスライドを生成する。

---

## コントリビュート

PRを歓迎します。最低限、以下を守ってください。

- 新規スキルは `skills/` 以下にフォルダを作成し、SKILL.md を配置（skill-creator のガイドラインに従う）
- SKILL.md は必須
- 機密情報・個人情報を含めない（ログ・スクショ含む）
- 既存スキルの変更は **後方互換** を意識（入力形式の破壊的変更は避ける）

---

## セキュリティ

- シークレット（APIキー等）はコミットしない
- 必要なら `.env.example` を置き、実キーは各自で管理
- 脆弱性/セキュリティ懸念は Issue または private 連絡（運用に合わせて）

## 運用メモ（任意）

- スキルは「増やす」より「使われる」ことを重視
- 使われないスキルは、理由が `SKILL.md` に残っていないことが多い（目的、手順、制約が曖昧）
- “万能スキル”は破綻しやすいので、目的は絞る（1スキル=1目的）
