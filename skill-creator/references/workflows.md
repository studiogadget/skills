# ワークフロー設計パターン

このファイルは、スキルの本文（SKILL.md）を「再現性が高い手順」にするための型をまとめています。

**GitHub Copilot 向けの注意**:
- ワークフローは **3〜7ステップ** を推奨（長すぎると読まれない）
- 各ステップは **1〜2行** で記述
- 複雑な判断が必要な場合は、別の参照ファイルに分離

## 逐次（シーケンシャル）ワークフロー

複雑なタスクは、操作を**明確な連続ステップ**に分解します。
SKILL.md の冒頭に、全体像（手順の列）を置くと、AIエージェントが迷いにくくなります。

```markdown
PDFフォームの入力は、次のステップで進めます:

1. フォームを解析（analyze_form.py を実行）
2. フィールド対応表を作成（fields.json を編集）
3. 対応表を検証（validate_fields.py を実行）
4. フォームを入力（fill_form.py を実行）
5. 出力を確認（verify_output.py を実行）
```

## 条件分岐（コンディショナル）ワークフロー

分岐があるタスクは、判断ポイントを明示し、AIエージェントがどの手順に入るべきかを案内します。

```markdown
1. 変更タイプを判定する:
   **新規作成？** → 下の「作成フロー」に従う
   **既存編集？** → 下の「編集フロー」に従う

2. 作成フロー: [steps]
3. 編集フロー: [steps]
```
